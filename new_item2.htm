DEFINIÇÃO

Nesta fase, integralmente executada de forma interna, é definido o âmbito do Projeto de Desenvolvimento do Produto / Release, documentados os seus requisitos, e executada a consulta a todos os stakeholders para registar a sua visão do âmbito.

 

OBJETIVO

Esta fase tem como objetivo assegurar todas as condições necessárias ao início do Projeto de Desenvolvimento do Produto / Release e aferir a sua viabilidade, no que diz respeito à mobilização dos recursos, preparação das condições logísticas e tecnológicas, e alinhamento de todos os intervenientes com os objetivos e âmbito do Projeto de Desenvolvimento do Produto / Release.

 

Nesta fase é definido e aprovado o calendário do Projeto de SI por todos os stakeholders com participação ativa no Projeto de Desenvolvimento do Produto / Release.        

 

O setup administrativo do Projeto de Desenvolvimento do Produto / Release deverá ter início antes da fase de Execução. Importante que, nenhum trabalho de execução, nomeadamente compromissos com custos, tenha início antes da aprovação definitiva do Projeto de Desenvolvimento do Produto / Release.

 

 

NOTA IMPORTANTE

Para se instanciar a framework de gestão de produto, temos que estar perante estas situações:

 

1. Proposta comercial adjudicada por um cliente que dará origem a um projeto de implementação liderado internamente pelas equipas de Operações (UN Corporativa) e Engenharia (UN da Efacec Transportes)

 

 

 

 

2. Change request solicitado pela equipa Engenharia (UN da Efacec Transportes) para um projeto de implementação em curso que dará origem a um projeto de desenvolvimento do produto

 

3. Change request solicitado pelo Gestor de Produto (UN da Efacec Transportes) para implementação de alterações previstas no Roadmap do produto
